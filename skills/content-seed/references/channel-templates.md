# チャネル別 構成テンプレート

## 共通の構成原則（6 ステップ）

どのチャネルでも以下の構造を基本とする：

1. **フック** — なぜこの話が聞く価値があるのか（聴衆の課題に接続）
2. **コンテキスト** — 何をしようとしていたか（状況設定）
3. **判断** — 何を選び、何を選ばなかったか
4. **根拠** — なぜそう判断したか
5. **結果** — やってみてどうだったか（成功も失敗も）
6. **学び / Takeaway** — 聴衆が持ち帰れるもの

---

## LT（5 分）

**特徴**: 1 メッセージ = 1 スライドの密度。結論ファースト。最後に CTA。

**スライド枚数の目安**: 6〜8 枚

```markdown
1. タイトルスライド
   - キャッチコピー（フック）
   - 登壇者の文脈を一言で

2. 課題提示（Who cares?）
   - 聴衆の誰がこの話を必要としているか

3. 検討した選択肢
   - A vs B vs C の比較軸を明示

4. 判断と根拠
   - 何を選んだか（1 スライドに絞る）
   - 決め手となった観点

5. 結果
   - やってみてどうだったか（数値・エピソード）

6. Takeaway + CTA
   - 「持ち帰ってほしい 1 文」
   - 続きを読むリンク / 質問歓迎の一言
```

---

## LT（15〜20 分）

**特徴**: ストーリーアークあり。デモや具体例を含む余裕がある。

**スライド枚数の目安**: 15〜20 枚

```markdown
1. タイトルスライド
2. アジェンダ（全体の見通し）
3. 背景・課題（2〜3 枚）
   - なぜこの判断が必要になったか
   - 当時のチームの状況
4. 選択肢の整理（2〜3 枚）
   - 検討した候補と評価軸
   - 各選択肢のトレードオフ
5. 判断と根拠（2〜3 枚）
   - 決定した内容
   - 不採用にした理由
6. 実装・導入（2〜3 枚）
   - 実際にやったこと
   - 躓いたポイント
7. 結果（2 枚）
   - 定量的・定性的な成果
   - 想定外だったこと
8. 学び・Takeaway（2 枚）
   - 汎用化した教訓
   - 次にやること
9. まとめ + CTA
```

## 技術ブログ

**特徴**: 見出し構成 + 各セクションの要点。コード例の挿入箇所を指定。

**文字数の目安**: 2,000〜5,000 字

```markdown
# [タイトル]（結論または問いかけ）

## TL;DR
- [3 点以内で要約]

## 背景
[なぜこの判断が必要になったか。チームの状況・制約を説明]

## 検討した選択肢
### A: [選択肢名]
- 概要
- 長所 / 短所

### B: [選択肢名]
- 概要
- 長所 / 短所

## 判断と根拠
[何を選び、なぜその判断をしたか。評価軸を明示]

```[言語]
// コード例の挿入箇所（実際のコードはユーザーが記載）
```

## 結果
[やってみてどうだったか。成功・失敗・想定外を含める]

## まとめ
[記事全体の要点を 3 点以内で再掲]

## 参考リンク
- [関連ドキュメント・記事]
```

## Twitter / X スレッド

**特徴**: フックの 1 投目が最重要。各ツイートが独立して読める構成。

**ツイート数の目安**: 5〜10 投

```
[1/n] フック（問いかけ or 逆説）
例: 「TCA、使いませんでした。理由を話します🧵」

[2/n] コンテキスト
例: 「iOS/Android 4 人チームでアプリを新規開発することになった。アーキテクチャどうする？」

[3/n] 検討した選択肢
例: 「候補は 3 つ: TCA / MVVM+Clean Arch / VIPER。評価軸は…」

[4/n 〜 n-2/n] 判断と根拠（1 つずつ展開）

[n-1/n] 結果
例: 「半年運用した結果…」

[n/n] Takeaway + CTA
例: 「学んだのは『流行ってるから』ではなく『チームの文脈で判断する』こと。ブログに詳しく書きました→[URL]」
```

## 社内共有（勉強会等）

**特徴**: 社内情報をそのまま含められる。抽象化不要。ディスカッション起点として使える。

```markdown
# [タイトル]

## 概要
[何を判断したか、一段落で]

## 背景と制約
[社内固有の文脈をそのまま記載可能]

## 検討した内容
[選択肢と評価]

## 判断と現在の状況
[決定内容と直近の状況]

## ディスカッションしたいこと
- [問いかけ 1]
- [問いかけ 2]
- [問いかけ 3]

## 参考
[関連 ADR・PR・ドキュメントのリンク]
```
