# スライド枚数・情報量ガイドライン

## 発表時間別スライド枚数と情報量の目安

| 発表時間 | スライド枚数目安 | 1枚あたり時間 | 話せる文字数（日本語） |
|---------|---------------|-------------|-------------------|
| 3分 | 3〜5枚 | 30秒〜1分 | 約900文字 |
| 5分（LT） | 5〜20枚 | 15秒〜1分 | 約1,500文字 |
| 10分 | 6〜12枚 | 1〜2分 | 約3,000文字 |
| 15分 | 10〜15枚 | 1〜1.5分 | 約4,500文字 |
| 20分 | 10〜20枚 | 1〜2分 | 約6,000文字 |

**話速の基準**: 日本語は1分間約300文字（落ち着いたペース）

## 5分LTのスタイル別スライド枚数

スタイルによって最適枚数は大きく異なる:

| スタイル | 枚数目安 | 特徴 |
|---------|---------|------|
| テキスト重視型 | 5〜8枚 | 内容密度が高く、各スライドをじっくり説明 |
| バランス型 | 10〜12枚 | 図表と文章を組み合わせた標準構成 |
| ビジュアル中心型 | 15〜20枚 | 画像・図解メイン、テンポよく進む |
| 高橋メソッド型 | 20〜30枚 | 1〜2語の超大文字、インパクト重視 |

## デザイン原則

### 1スライド1メッセージ（必須ルール）
- 各スライドで伝えることは**1つだけ**
- スライドタイトルがそのメッセージになるよう書く
- 「〇〇について」より「〇〇が重要な理由は△△だから」が良いタイトル

### Guy Kawasakiの10/20/30ルール（参考値）
- スライドは**10枚以内**
- 発表は**20分以内**
- フォントは**30pt以上**

### 6×6ルール（情報密度の上限）
- 1スライドに**6行以下**
- 1行は**6語以下**（英語基準。日本語は1行15〜20文字以内が目安）

## 標準的なスライド構成テンプレート

### 5分LT（10枚構成例）
```
1. タイトル + 自己紹介（30秒）
2. 結論・今日伝えたいこと（30秒）
3. 背景・課題設定（30秒）
4-7. 本論（各30秒〜1分）
8. まとめ・学び（30秒）
9. 次のアクション・呼びかけ（30秒）
10. 質疑応答・連絡先（残り時間）
```

### 10分発表（12枚構成例）
```
1. タイトル + アジェンダ（1分）
2. 自己紹介・文脈（1分）
3. 問題提起・背景（1分）
4-9. 本論（各1〜1.5分）
10. まとめ（1分）
11. 質疑応答（残り時間）
```

## ビジュアルタイプの選び方

| 伝えたいこと | 推奨ビジュアルタイプ |
|------------|-------------------|
| 数値・実績 | グラフ、数字の強調表示 |
| 比較・対比 | テーブル、2カラムレイアウト |
| 流れ・プロセス | フローチャート、ステップ図 |
| 関係性・構造 | 概念図、マインドマップ |
| 時間軸 | タイムライン |
| 強いメッセージ | テキストのみ（高橋メソッド的） |
| 雰囲気・共感 | 写真・イラスト |
