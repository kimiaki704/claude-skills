## 各フレームワークの詳細

### PREP法（最汎用）
**Point → Reason → Example → Point**

結論を冒頭と末尾で2回伝えることで記憶に残りやすい。LTの5分間で最もよく使われる。

```
[オープニング] 結論（Point）を一言で
[本論] なぜそうなのか（Reason）
       実例・データ（Example）
[クロージング] 結論を再提示（Point）
```

### 結起承転結（日本LT標準）
**結論 → 起点（背景） → 展開 → 転換 → 結論**

従来の起承転結を「結論ファースト型」に変形。技術コミュニティで特に推奨。

```
[冒頭] 今日言いたいこと（結論）
[起] 背景・きっかけ
[承] 経緯・詳細
[転] 山場・発見・失敗
[結] 結論の再提示 + 次のアクション
```

### SDS法（報告・共有向け）
**Summary → Details → Summary**

「ストーリー全体の理解」を重視。短時間セミナーや進捗報告に最適。

```
[冒頭] 全体サマリー（Summary）
[本論] 詳細説明（Details）
[締め] サマリーの再提示（Summary）
```

### SCR / SCQA（コンサル系・問題解決）
**Situation → Complication → Resolution / Question → Answer**

バーバラ・ミントのピラミッド原則に基づく。問題意識が明確な聴衆向け。

```
[現状] Situation: 今何が起きているか
[複雑化] Complication: しかし問題がある
[解決] Resolution: だからこうする
（SCQA版は Complication の後に Question を挿入）
```

### TAPS法（現状→理想ギャップ型）
**To Be → As Is → Problem → Solution**

理想と現状のギャップを問題として提示する。変革提案に有効。

```
[理想] To Be: こうあるべき姿
[現状] As Is: 実際はこうなっている
[問題] Problem: ギャップの正体
[解決] Solution: 提案するアプローチ
```

### FABE法（営業・製品紹介）
**Feature → Advantage → Benefit → Evidence**

製品・サービス・技術の紹介に特化。聴衆への具体的な価値訴求が目的。

```
[特徴] Feature: どんな機能・仕様か
[優位性] Advantage: 他と何が違うか
[便益] Benefit: 聴衆にどんな価値があるか
[証拠] Evidence: 事実・データ・実績
```

### ストーリーアーク（ナラティブ型）
**Exposition → Rising Action → Climax → Falling Action → Resolution**

時系列・物語構造。体験談・事例紹介に強い。

```
[導入] Exposition: 日常の状況設定
[展開] Rising Action: 問題や挑戦の積み上げ
[クライマックス] Climax: 最大の山場・転換点
[収束] Falling Action: 解決へ向かう流れ
[結末] Resolution: 現在の状態・学び
```

### Nancy Duarteスパークライン（変革・感動型）
**「現状 What Is」と「理想 What Could Be」のコントラストを繰り返す**

スティーブ・ジョブズのiPhone発表で実証済み。聴衆の感情を揺さぶる。

```
[現状] What Is: 現実の不満・課題
[理想] What Could Be: もしこうなったら
[現状] What Is: さらに大きな問題
[理想] What Could Be: でもこういう世界が
...（繰り返し）
[締め] New Bliss: 新しい世界への呼びかけ
```
