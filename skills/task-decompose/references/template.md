# 起票テンプレート

Step 6 で壁打ち結果を整形する際に使用するテンプレート。

<!-- keep-ja -->
```markdown
## 背景（なぜ）

このタスクが必要な理由。プロジェクトのどの部分に影響するか。
これがないと何が困るか。

## ゴール（何を）

このタスクが完了したとき、何が達成されている状態か。
成果物は何か（ドキュメント、PR、動作するコード、提案書、ADR など）。

## 完了条件

- [ ] 条件1
- [ ] 条件2
- [ ] 条件3

## スコープ（やること・やらないこと）

**やること:**
- xxx

**やらないこと（明示的に除外）:**
- xxx

## 期日

- 期日: YYYY-MM-DD
- 期日の根拠: （例: この結果をもとに次のスプリントで実装に入るため）

## 確認ポイント

- [ ] 着手時: 方針が合っているか確認（必要なら壁打ち）
- [ ] 中間: （例: 調査結果が出た時点で一度共有）
- [ ] 完了時: 完了条件を満たしているかセルフチェック・共有先に報告

## 責任

- オーナー（見届ける人）: このタスクが止まっていたら突く人。1人だけ指定する
- 担当者（実行する人）: 実際に手を動かす人。未定なら「未定」でOK。`gh issue create --assignee` に変換される
- 相談先（任意）: 判断に迷ったときの壁打ち相手
- 共有先（任意）: 完了時に結果を知るべき人

## メタ情報（任意）

- 優先度: P0（クリティカル）/ P1（高）/ P2（中）/ P3（低）
- ラベル: （`gh issue create` 時に `--label` に変換）
- プロジェクト: （`gh issue create` 時に `--project` に変換）

## 参考情報

- 関連 issue / PR
- 参考ドキュメント・リンク
- 関連する過去の意思決定
```
