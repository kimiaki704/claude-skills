# 起票テンプレート

Step 6 で壁打ち結果を整形する際に使用するテンプレート。

<!-- keep-ja -->
```markdown
## 背景（なぜ）

このタスクが必要な理由。プロジェクトのどの部分に影響するか。
これがないと何が困るか。

## ゴール（何を）

このタスクが完了したとき、何が達成されている状態か。
成果物は何か（ドキュメント、PR、動作するコード、提案書、ADR など）。

## 完了条件

- [ ] 条件1
- [ ] 条件2
- [ ] 条件3

## スコープ（やること・やらないこと）

**やること:**
- xxx

**やらないこと（明示的に除外）:**
- xxx

## 期日

- 期日: YYYY-MM-DD
- 期日の根拠: （例: この結果をもとに次のスプリントで実装に入るため）

## 確認ポイント

- [ ] 着手時: 方針が合っているか確認（必要なら壁打ち）
- [ ] 中間: （例: 調査結果が出た時点で一度共有）
- [ ] 完了時: 完了条件を満たしているかセルフチェック

## メタ情報（任意）

- 担当者: （`gh issue create` 時に `--assignee` に変換）
- ラベル: （`gh issue create` 時に `--label` に変換）
- プロジェクト: （`gh issue create` 時に `--project` に変換）

## 参考情報

- 関連 issue / PR
- 参考ドキュメント・リンク
- 関連する過去の意思決定
```
<!-- /keep-ja -->
