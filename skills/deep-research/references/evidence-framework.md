# エビデンスフレームワーク

## 事実のエビデンスレベル（E1-E4）

ファインディングスの各主張に付与する。調査のすべての Phase で一貫して使用する。

| レベル | ラベル | 定義 | 具体例 |
|--------|--------|------|--------|
| **E1** | 一次ソース | 公式ドキュメント、査読付き論文、RFC、公式ベンチマーク、公式リリースノート | RFC 7540 (HTTP/2)、Apple Developer Documentation、arXiv 掲載論文 |
| **E2** | 信頼できる二次ソース | 企業エンジニアリングブログ、著名 OSS のドキュメント、カンファレンス発表資料、業界レポート | Netflix Tech Blog、WWDC セッション、ThoughtWorks Technology Radar |
| **E3** | 一般的な二次ソース | 個人ブログ、Stack Overflow / フォーラムの回答、ニュース記事、チュートリアル | Medium 記事、Zenn 記事、Hacker News のコメント |
| **E4** | 未検証 | 単一ソースのみで裏付けなし、出典不明、伝聞、SNS の投稿 | 「〜らしい」「一般的に〜と言われている」 |

### E レベル判定ルール

1. **ソースが明記されている場合**: ソースの種類で E1-E3 を判定
2. **ソースが明記されていない場合**: E4 とする
3. **複数ソースで裏付けられている場合**: 最も信頼性の高いソースの E レベルを採用
4. **ソース間で矛盾がある場合**: 両方のエビデンスを提示し、矛盾を明示する。一方的に片方を採用しない
5. **公式ドキュメントの日付が古い場合**: E1 だが「{年}時点の情報」と注記する
6. **翻訳・要約を介している場合**: 原文のレベルから 1 段階下げて判定する（E1 → E2）

### E4 の扱い

E4 の主張はファインディングスにそのまま混ぜない。以下のいずれかの形で提示する:

- 「**未検証**: 〜という主張がある（要深掘り）」としてギャップマップに記載
- 深掘り候補として Phase 4 の推奨リストに含める
- 複数の E4 主張が同じ方向を示している場合: 「複数の未検証ソースが 〜 を示唆しているが、一次ソースでの確認が必要」

## 推論レベル（P1-P4）

未来シナリオ分析で使用する。事実（E レベル）に基づく推論の蓋然性を示す。

**重要**: P レベルの推論には、根拠となる E レベルの事実が必ず紐づいている必要がある。根拠のない推論は提示しない。

| レベル | ラベル | 定義 | 必要な根拠 |
|--------|--------|------|-----------|
| **P1** | 高蓋然性 | 複数の独立したトレンドが同じ方向を示す。メカニズムが明確 | E1-E2 の事実が 3 つ以上、因果関係のメカニズムが説明可能 |
| **P2** | 蓋然性あり | トレンドは存在するが、阻害要因も確認されている | E1-E2 の事実が 2 つ以上、ただし反証・阻害要因も存在 |
| **P3** | 可能性 | 論理的に成立するが、裏付けるトレンドが弱いまたは少ない | E2-E3 の事実が 1-2 つ、または間接的な根拠のみ |
| **P4** | 思考実験 | 前提を変えた場合の仮説。現時点のトレンドとは異なる方向 | 「もし X が起きたら」という仮定を明示。現在の事実と乖離している点を記載 |

### P レベル判定ルール

1. **根拠の E レベルが低い場合**: P レベルも自動的に低くなる。E4 のみに基づく推論は P4 とする
2. **根拠が複数の独立したソースから来ている場合**: P レベルを 1 段階引き上げてよい
3. **予測の時間軸が長い場合**: 不確実性が増すため P レベルを 1 段階引き下げる（目安: 3 年以上先 → 1 段階下げ）
4. **阻害要因が存在する場合**: P1 → P2 に下げ、阻害要因を明記する

### P レベルの表記

シナリオを提示する際、以下の形式で P レベルと根拠を紐づける:

```
シナリオ A: [名前] (P1: 高蓋然性)
  根拠:
  - [E1] トレンド X: (内容) — ソース: (URL)
  - [E2] トレンド Y: (内容) — ソース: (URL)
  - [E1] メカニズム: X と Y が Z を引き起こす理由 — ソース: (URL)
  帰結: ...
```

## E レベルと P レベルの関係

```
事実の世界（過去・現在）        推論の世界（未来）
┌─────────────────┐        ┌─────────────────┐
│ E1: 一次ソース    │──┐     │ P1: 高蓋然性     │
│ E2: 信頼二次ソース │──┼──→ │ P2: 蓋然性あり   │
│ E3: 一般二次ソース │──┘     │ P3: 可能性       │
│ E4: 未検証       │───×    │ P4: 思考実験     │
└─────────────────┘        └─────────────────┘
                             ※ E4 のみでは P レベルの
                               推論を生成しない
```

E レベルは「何が事実か」を示し、P レベルは「事実からどこまで推論できるか」を示す。
この分離により、ユーザーは「ここまでは確かな事実、ここからは推論」を常に判断できる。
